#Abstract 

This report details the quality assessment of pre-injection baseline data from 11 High-Performance Liquid Chromatography (HPLC) runs. The primary objective was to verify column integrity, segment baseline regions, and quantify signal stability using statistical metrics and visual analysis. Critical findings include the identification of a mislabeled data column, a significant metadata anomaly regarding sample identifiers, and a systemic violation of noise thresholds across all runs. While no runs violated physical detector limits, Run 32 exhibited extreme instability, and all runs demonstrated non-zero drift slopes requiring correction. The analysis confirms that raw data contains substantial noise and non-linear drift components, necessitating the application of advanced baseline correction models prior to quantitative integration.

#Introduction 

Accurate quantitative analysis in HPLC relies heavily on the stability of the pre-injection baseline. Systematic artifacts such as constant offsets, linear drift, and non-linear trends can significantly compromise peak integration and subsequent concentration calculations. This study focuses on evaluating the raw sensor signals (UV_1_280_mAU and UV_2_260_mAU) within the pre-injection region (volume < 0 mL) across 11 distinct chromatography runs. The motivation for this analysis is to validate the equilibrium stability of the detector, identify data integrity issues such as mislabeled columns or metadata inconsistencies, and determine if the observed noise and drift exceed regulatory standards (USP <621>) or physical detector limits. By quantifying these artifacts, we aim to establish robust rejection criteria and define appropriate correction models, such as linear or polynomial fitting, to ensure data integrity before downstream processing.

#Methodology 

The data analysis workflow comprised three sequential steps. First, Column Integrity Verification and Baseline Data Segmentation were performed on the combined dataset. This involved calculating the correlation between 'UV_1_280_ml' and 'volume_ml' to identify mislabeled duplicates and filtering rows where 'volume_ml' < 0 to isolate the baseline region. Metadata consistency was validated by checking the missingness rates of 'Sample_Code' and 'Fraction_unique_ID'. Second, Run-Specific Baseline Stability & Drift Profiling was conducted for each of the 11 runs. Linear regression was applied to the baseline data to calculate drift slopes and signal offsets, while Residual Standard Deviation (RSD) was computed to assess noise floors. A second-order polynomial fit was also generated to detect non-linear trends, comparing the R² improvement against the linear model. Finally, Visual Baseline Trend Analysis was executed for the first five runs to provide graphical evidence of the statistical findings, overlaying raw data with linear and polynomial fits and highlighting physical limit violations.

#Results and Discussion 

The initial data integrity check revealed a critical structural error: the column 'UV_1_280_ml' showed a perfect correlation (1.0000) with 'volume_ml', confirming it as a mislabeled duplicate of the volume axis rather than a UV signal. Consequently, this column was excluded from all signal analyses. While the baseline segmentation successfully identified between 1,804 and 6,360 rows per run, a significant metadata anomaly was detected. The analysis expected approximately 75% missingness in sample identifiers for the baseline region; however, the observed missingness was 99.99%. This deviation suggests a systemic data entry error or an overly broad segmentation logic, preventing immediate validation of the baseline segment as intended.

Regarding signal stability, the analysis of the 11 runs revealed a systemic issue with noise levels. Every run exhibited a 'True' status for Noise_Threshold_Violation, indicating that the baseline noise (RSD) exceeded the regulatory threshold (likely > 0.5% per USP <621>). Run 32 emerged as a severe outlier, displaying extreme instability with UV_260 RSD of 819.47 and UV_280 RSD of 42.05, suggesting potential sensor malfunction or data corruption. In contrast, Run 18 demonstrated the most stable baseline with low RSD values (0.41 and 0.56). Despite the high noise, no runs violated physical detector limits (signal < -10 mAU), although negative offsets were observed (e.g., -0.75 mAU in Run 16).

As illustrated in Figure 1, the visual analysis of the first five runs confirms the statistical findings of systematic drift. The raw data exhibits clear non-zero slopes, indicating linear drift across the baseline period. Furthermore, the second-order polynomial fits track the curvature of the raw data more closely than the linear models in several instances, suggesting the presence of non-linear drift components. The visualization also highlights regions where signal values approach the -10 mAU physical limit, validating the necessity of the proposed baseline correction protocols. The combination of high noise floors, systematic drift, and non-linear trends confirms that raw data cannot be used for quantitative integration without applying run-specific correction models, particularly for the highly unstable Run 32 which requires immediate investigation or exclusion.

#Conclusion 

The analysis of the HPLC pre-injection baseline data has identified significant data quality issues that must be addressed before quantitative analysis. A mislabeled column was successfully identified and removed, but a pervasive metadata anomaly regarding sample identifiers requires further investigation to ensure correct baseline segmentation. Most critically, all 11 runs failed the noise threshold validation, with Run 32 exhibiting catastrophic instability. The presence of both linear and non-linear drift, as confirmed by statistical metrics and visual inspection in Figure 1, necessitates the implementation of robust baseline correction algorithms. Future workflows must incorporate these correction models and address the metadata inconsistencies to ensure the physical validity and accuracy of the chromatographic data.
