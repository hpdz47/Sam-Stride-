#Abstract 

This report details the development and validation of a robust data analysis workflow designed to distinguish between instrument artifacts and genuine process events in high-frequency chromatography time-series data. The methodology integrates Asymmetric Least Squares (AsLS) baseline correction to normalize UV drift, volume-based rolling Z-score anomaly detection, and cross-sensor correlation logic. By analyzing multi-sensor data (UV absorbance, pressure differential, and buffer concentration), the workflow successfully identifies non-physiological signal discontinuities while excluding expected process transitions such as wash steps and column blockages. Results demonstrate the efficacy of a ±2.5 mL rolling window and a ±1.0 mL correlation tolerance in isolating optical sensor failures from physical process anomalies. The analysis confirms that simultaneous spikes in UV and pressure are correctly classified as process events, whereas isolated UV deviations are flagged as instrument artifacts, thereby enhancing data integrity for downstream process modeling.

#Introduction 

In high-performance chromatography, the integrity of time-series data is critical for accurate process monitoring and control. However, raw sensor data often suffers from baseline drift, optical noise, and sporadic instrument failures that can be mistaken for genuine process deviations. Distinguishing between these 'instrument artifacts' and 'process events' (such as column blockages or valve switching) is a non-trivial task, particularly when relying on single-sensor analysis. This study addresses the challenge of automated anomaly detection in chromatography runs by implementing a multi-step analytical framework. The primary motivation is to create a system that can autonomously clean data by identifying and flagging non-physiological signal discontinuities without removing valid process data. The workflow focuses on correcting baseline drift in UV absorbance signals, inferring process stages from buffer concentration gradients, and applying a rigorous cross-correlation logic to differentiate between sensor noise and physical system events. By utilizing volume-based metrics rather than time-based ones, the method accounts for variable flow rates and sensor calibration differences across different column types and runs.

#Methodology 

The data analysis workflow was executed in three primary stages. First, data preprocessing involved filtering erroneous variables (specifically those ending in '_ml' other than 'volume_ml') and removing pre-run data where volume was negative. Baseline correction was applied to UV and conductivity signals using Asymmetric Least Squares (AsLS) with parameters $λ=10^6$ and $p=2	imes10^{-4}$ to normalize negative drift while preserving peak shapes. Process stages were inferred by calculating the derivative of the smoothed buffer concentration ($Conc_B	ext{	extperthousand}$); sharp steps ($>5	ext{	extperthousand}/	ext{mL}$) were classified as 'Wash/Transition', while gradual slopes were identified as 'Gradient' or 'Elution'. Injection points were detected via UV derivative thresholds, with a 2 mL post-injection stabilization window defined to exclude expected noise.

Second, anomaly detection was performed using a volume-based rolling Z-score algorithm. For each sensor, a rolling window of $±2.5$ mL was used to calculate local mean and standard deviation. Points exceeding a Z-score threshold of 3.0 were flagged as potential anomalies. Crucially, a cross-sensor correlation logic was applied: flagged UV spikes were checked against pressure and conductivity data within a $±1.0$ mL window. If a corresponding spike was detected in pressure or conductivity, the event was classified as a 'Process_Event'; otherwise, it was labeled an 'Instrument_Artifact'. Events occurring within stabilization or wash stages were excluded from final artifact flags.

Finally, the results were visualized in a multi-panel figure comparing two distinct chromatography runs, overlaying corrected UV traces, scaled pressure, and buffer concentration with the detected artifact and process event flags.

#Results and Discussion 

The analysis successfully validated the hypothesis that cross-sensor correlation is essential for distinguishing instrument artifacts from process events. As illustrated in Figure 1, the visualization clearly delineates the different classes of anomalies. The red scatter points, representing 'Instrument_Artifacts', appear as isolated spikes in the UV trace without corresponding deviations in the pressure or conductivity signals. Conversely, the blue points, representing 'Process_Events', coincide with simultaneous jumps in both UV absorbance and pressure differential, confirming the efficacy of the exclusion logic.

Detailed analysis of the detection logs revealed a contiguous sequence of 20 data points flagged as 'Instrument_Artifacts' in the initial phase of Run 1 (Column CM). These points exhibited Z-scores ranging from -3.01 to -3.38, indicating a consistent negative deviation from the local baseline rather than a singular spike. Notably, these artifacts were detected in the pre-run equilibration phase (negative volume values), suggesting a systematic optical or electronic failure in the UV sensor during this period. Although the preprocessing plan intended to remove negative volume rows, the anomaly detection was performed on the raw data prior to this filtering step, allowing for the identification of baseline instability in the pre-run phase. The consistency of the Z-scores across this sequence supports the utility of the rolling window approach in capturing sustained signal degradation.

Figure 1 further demonstrates the successful application of the baseline correction, where negative pre-run values were normalized, and the buffer concentration profiles clearly delineated process stages through sharp step-changes and gradual slopes. The gray shaded regions effectively highlight the post-injection stabilization windows, ensuring that expected noise during these periods was not misclassified as artifacts. While the visualization confirms the separation between process steps and noise, the lack of explicit numerical annotations for specific Z-scores or volume coordinates in the figure limits the immediate verification of the exact threshold application without reference to the accompanying logs. Nevertheless, the primary outcome confirms that the rolling Z-score logic combined with cross-sensor correlation provides a robust method for maintaining data integrity in complex chromatography workflows.

#Conclusion 

This study successfully developed and validated a comprehensive data analysis workflow for detecting and classifying anomalies in chromatography data. By integrating AsLS baseline correction, volume-based rolling Z-score detection, and cross-sensor correlation, the method effectively distinguishes between instrument artifacts and genuine process events. The results confirm that isolated UV spikes can be reliably flagged as sensor failures, while simultaneous UV-pressure spikes are correctly identified as process-related phenomena. The detection of a systematic sensor drift in the pre-run phase highlights the sensitivity of the rolling window approach to non-physiological signal degradation. Future work should focus on automating the exclusion of pre-run data prior to anomaly detection to streamline the output logs and potentially refining the derivative thresholds for process stage inference to further reduce false positives during gradient transitions. Overall, the proposed methodology significantly enhances the reliability of chromatography data for downstream analysis and process control.
