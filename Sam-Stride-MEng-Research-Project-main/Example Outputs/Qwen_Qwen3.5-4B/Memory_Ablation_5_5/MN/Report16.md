#Abstract 

This report evaluates the data integrity and suitability of a high-resolution HPLC chromatographic dataset for downstream machine learning applications. The primary objective was to assess signal quality, completeness, and unit consistency against strict biopharmaceutical standards. Analysis revealed critical data integrity failures, including significant missingness exceeding 65% in key variables such as concentration and pH, and the presence of physically impossible negative absorbance values in UV signal columns. These artifacts were identified through histogram distribution analysis and correlation checks against volume data. The study concludes that while the dataset contains robust primary signal trends, aggressive imputation is prohibited for high-missingness variables to prevent model bias. A final 'Fit for Use' dataset was derived by strictly filtering for variables with >95% completeness and valid units, resulting in a reduced feature set suitable for specific modeling linkages.

#Introduction 

In the biopharmaceutical manufacturing sector, the reliability of chromatographic records is paramount for ensuring product quality and process consistency. However, the integration of these records into predictive machine learning models requires rigorous validation of data integrity. This study focuses on a dataset comprising high-resolution fraction-level time-series data from HPLC runs. The data exhibits several critical integrity issues, including substantial missingness in process variables (e.g., concentration, pH), metadata gaps (e.g., Sample_Code), and unit mislabeling manifesting as negative absorbance values in physically constrained variables. The motivation for this analysis is to determine whether the dataset meets the strict thresholds required for feature engineering and predictive modeling. Specifically, the analysis aims to quantify the feasibility gap for sample-specific modeling and to establish a protocol for excluding corrupted variables rather than introducing bias through imputation. The ultimate goal is to provide a quantified decision on modeling feasibility and to guide the selection of a high-completeness variable subset.

#Methodology 

The data analysis workflow consisted of three sequential steps designed to audit, validate, and derive a final feature set. First, a Data Integrity and Completeness Audit was performed on the raw `chromatography_combined.csv` file. This step involved checking for missingness thresholds (>95% exclusion) and identifying unit validity issues, specifically negative values in variables containing physical quantities like absorbance. Second, a Chromatographic Quality and Signal Validation was conducted. This involved visualizing UV absorbance distributions to detect negative value artifacts and correlating `UV_1_280_mAU` with cumulative eluent volume (`volume_ml`) to identify potential mislabeling of units or baseline noise. Third, a High-Completeness Feature Set Derivation was executed. This step filtered the dataset to retain only variables meeting the >95% completeness threshold and valid unit ranges, explicitly excluding variables with high missingness or naming inconsistencies. Throughout the process, the analysis strictly prohibited aggressive imputation for variables failing completeness checks, ensuring that only data meeting rigorous biopharmaceutical standards were utilized for downstream modeling.

#Results and Discussion 

The analysis of the chromatographic dataset identified significant data integrity challenges that impact the reliability of downstream machine learning models. As illustrated in Figure 1, the histogram of UV absorbance signals (`UV_1_280_mAU` and `UV_2_260_mAU`) reveals a heavy concentration of data points near zero with a long tail into positive values. Crucially, the presence of a distinct cluster of points to the left of the zero line confirms the existence of negative absorbance artifacts. These values are physically impossible for absorbance and suggest baseline noise, integration errors, or unit mislabeling. The scatter plot in Figure 1, which correlates `UV_1_280_mAU` with `volume_ml`, further elucidates the nature of these errors. The data points form a dense, linear cloud with a positive slope, indicating a strong correlation between cumulative eluent volume and UV signal intensity. However, the negative values identified in the histogram correspond to extremely low (near-zero) volume readings, likely representing baseline noise or empty fraction collections rather than actual negative volumes. This visual evidence confirms that while the primary signal trend is robust, the dataset contains specific, quantifiable outliers that must be addressed. Furthermore, the audit revealed that key variables such as `Conc_B_ml` and `pH_ml` suffer from missingness exceeding 65%, while metadata variables like `Sample_Code` show even higher gaps. Given these findings, the methodology successfully identified that the dataset is not currently 'fit for use' in its raw form. To proceed, the dataset was filtered to retain only variables with >95% completeness and valid units, resulting in a reduced feature set of eight variables. This strict filtering approach ensures that the predictive models built upon this data are not biased by missing values or physically impossible measurements, thereby maintaining the integrity of the biopharmaceutical process data.

#Conclusion 

In conclusion, this study has rigorously evaluated the integrity of a high-resolution HPLC chromatographic dataset for use in machine learning applications. The analysis confirmed the presence of critical data quality issues, including high levels of missingness in process and metadata variables and the existence of negative absorbance artifacts indicative of data entry or measurement errors. The correlation analysis between UV signals and volume successfully validated the physical plausibility of the primary signal trends while isolating baseline noise as the source of negative values. The proposed solution involves a strict exclusion protocol, retaining only variables with >95% completeness and valid units, and explicitly prohibiting aggressive imputation for high-missingness variables. This approach ensures that the resulting 'Fit for Use' dataset is free from bias and meets the stringent biopharmaceutical standards required for reliable predictive modeling. Future work should focus on implementing the derived feature set for sample-specific modeling linkages and further investigating the root causes of the high missingness in concentration and pH variables.
