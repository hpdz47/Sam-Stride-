#Abstract 

This report evaluates the integrity of High-Performance Liquid Chromatography (HPLC) chromatographic data, specifically addressing severe data quality challenges including missing metadata, negative volume values, and high signal variance. The primary objective is to determine whether robust normalization or dynamic baseline adjustment is required to mitigate process instability. Analysis reveals a critical discrepancy between the data analysis plan and the execution results: the initial data integrity check reported zero filtered rows, contradicting the existence of massive missingness (~75%) in metadata columns and negative volume values identified in the Exploratory Data Analysis (EDA). Consequently, the reported Signal-to-Noise Ratio (SNR) values and subsequent conclusions regarding baseline correction are likely based on an unfiltered, full dataset containing invalid records. This report concludes that the current analysis phase has failed to isolate process instability or exclude physical anomalies, thereby undermining the validity of the SNR metrics and the proposed baseline correction strategies. Rigorous filtering and re-evaluation of the dataset are required before proceeding to normalization strategy evaluation and calibration curve construction.

#Introduction 

The dataset under investigation represents a multi-stage purification process characterized by significant data quality issues. Specifically, the UV signal columns exhibit high variance (standard deviation > 390 mAU), which may mimic baseline shifts, while critical quantitative variables such as `Conc_B_ml` are missing in over 65% of records. Furthermore, the metadata columns `chromatography_stage` and `Sample_Code` suffer from massive missingness (~75%), and the `volume_ml` variable contains negative values representing physical anomalies. The motivation for this data analysis workflow is to rigorously evaluate data integrity to determine if robust normalization (using `_min` flow variables) or dynamic baseline adjustment (per `run_no`) is required to mitigate process instability, rather than relying on static baseline correction. The primary goal is to assess if high UV signal variance stems from instrument noise or process drift, specifically accounting for the severe missingness in concentration and pH data. The analysis must define quantitative interpretation by constructing calibration curves using `UV_1_280_mAU` peak areas against available concentration points to establish response factors for proxy quantification. Furthermore, the analysis must establish a strategy for handling physical anomalies and exclude unreliable metadata to ensure unbiased variance decomposition between process conditions and instrument noise.

#Methodology 

The data analysis plan was structured into three sequential steps designed to address data integrity, normalization strategies, and calibration curve construction. 

Step 1: Data Integrity & Anomaly Filtering. The objective was to load the `chromatography_combined.csv` dataset and filter rows based on specific criteria: `chromatography_stage` set to 'Preparation' or 'Cleanup', `Sample_Code` set to 'Blank' or 'Control', and `volume_ml` < 0. The plan required calculating the Signal-to-Noise Ratio (SNR) for `UV_1_280_mAU` and `UV_2_260_mAU` using the standard deviation of the signal divided by the median absolute deviation (MAD) of the signal during the stable baseline period (the first 10% of `volume_ml`), while avoiding low-variance `_min` flow variables. The expected output included a count of filtered rows, summary statistics for UV signals and conductivity, and average SNR per run number.

Step 2: Normalization Strategy Evaluation. This step aimed to implement two distinct strategies to address signal variance: 1) Static baseline correction (subtracting the mean of the run), and 2) Dynamic normalization using the mean of `System_flow_ml_min` for the specific `run_no` to scale UV signals. The methodology involved addressing missingness in `pH_ml` by excluding rows with missing values prior to calculations. The comparison focused on the variance of normalized signals between methods.

Step 3: Calibration Curve Construction for Proxy Quantification. This final step involved filtering rows where `Conc_B_ml` was missing and handling missingness in `pH_ml` by dropping rows containing any NaN values. The `UV_1_280_mAU` signal was aggregated to the `run_no` level (mean signal) and fitted against `Conc_B_%` using linear regression. The output was intended to provide the regression equation, R-squared value, standard error of estimate, and a statistical significance summary.

#Conclusion 

The current data analysis workflow has encountered a critical failure in the initial data integrity phase. While the analysis plan explicitly required filtering out rows with negative volumes and unreliable metadata (missing chromatography stage or sample code), the resulting report indicates that zero rows were filtered. This null result is critically inconsistent with the Exploratory Data Analysis (EDA) findings, which identified massive missingness (~75%) in the specified metadata columns and the presence of negative volume values. This discrepancy suggests that the filtering logic either failed to execute correctly or the data source for the report does not reflect the actual dataset state. Consequently, the subsequent summary statistics and SNR calculations presented in the report are likely based on the unfiltered, full dataset, potentially including invalid records and physical anomalies. This undermines the validity of the reported SNR values and the subsequent conclusions regarding baseline correction. Until the data integrity issue is resolved and the dataset is properly cleaned to exclude unreliable metadata and physical anomalies, the analysis cannot proceed to the normalization strategy evaluation or calibration curve construction with confidence. Future work must prioritize debugging the filtering step and re-running the analysis on a cleaned dataset to ensure unbiased variance decomposition.
