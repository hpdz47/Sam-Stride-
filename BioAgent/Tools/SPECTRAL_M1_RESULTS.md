# M1 ŌĆö Spectral tools vs. published ground truth

This milestone reimplements the *Dai et al., Nature 2024* hostŌĆōguest decision-maker
as **pure, deterministic, off-instrument tools** and validates them against the
paper's own published data + expected outputs (the `data/` bundle in the
`cooper-group-uol-robotics` repo, workflow **SUPRAMOL-SCREENING**, 18 samples =
3 amines ├Ś 3 carbonyls ├Ś 2 metals).

Design principle: the paper's decision-maker is *hand-coded heuristics*, not an
LLM. The MAS will **call, interpret, and report on** these validated tools; it
does not regenerate the numerics each run. The verdicts come from the tools; the
LLMs orchestrate and narrate.

## What was built

| File | Role |
|------|------|
| `Tools/Spectral_Analysis.py`   | Pure ports of the four decision heuristics + peak matching. No TopSpin, no Waters SDK, no I/O. |
| `Tools/Spectral_Ingestion.py`  | `nmrglue`+`scipy` replacement for the TopSpin processing/peak-pick layer. Reads Bruker **and** Varian; two paths (processed `1r`, or raw FID). |
| `Tools/tests/validate_screening.py`  | Decision-layer gate vs published `SUMMARY_NMR.json` / `SUMMARY_MS.json`. |
| `Tools/tests/validate_ingestion.py`  | Ingestion gate: picked peaks ŌåÆ decision rule ŌåÆ verdicts, both ingestion paths. |

### Decision ports (`Spectral_Analysis.py`), each a faithful copy of the reference
- `nmr_screening_rule` ŌåÉ `decisions/different_from_reagents.py` (`peak_number=3`, `0.5├Ś` new-peak test, 2-dp binning)
- `ms_quorum_rule` ŌåÉ decision half of `decisions/expected_mass_metals.py` (`metals_mz=[3,2]`)
- `match_ms_hits` ŌåÉ portable half of `lcms_parser.identify_hits` (`atol=0.4` Da, ES+)
- `replication_check` ŌåÉ `decisions/same_as_reference2.py` (DTW, prune `>0.1`, PASS `< 20.0`)
- `binding_shift_detection` ŌåÉ `decisions/guest_binding.py` (`np.isclose(atol=0.02)` ppm)
- `combine_screening` = `NMR_PASS AND MS_PASS`
- `match_peaks` = Hungarian min-|╬ö╬┤| (diagnostic / predict-then-compare channel)

## Validation results

**Decision layer** (fed the published peak lists / m/z hits):
```
NMR screening: 18/18 verdicts match
MS  screening: 18/18 verdicts match
```

**Ingestion layer** (peaks picked from the raw spectra, then run through the rule):
```
processed-1r path (reproduces the paper using TopSpin's output):
    peak recall 212/216 (98.1%), extra 2, missing 4
    18/18 NMR verdicts match
raw-FID path (fully portable, nmrglue only ŌĆö the path that reaches Varian):
    16/18 NMR verdicts match
```

The 2-verdict gap on the raw-FID path is phase/baseline difference vs. TopSpin's
proprietary AI phasing (`apbk`); it is the plan's risk #2 made concrete. For the
Bruker paper-reproduction we use the processed `1r` (18/18); for the Varian
dataset (M4) the raw-FID path is the only option and 16/18 on Bruker is the
confidence baseline to improve on.

## Notes / scope
- **MS raw parsing** (`.RAW` ŌåÆ hits) stays vendor-locked to the Waters MassLynx
  SDK (Windows-only). We validate the **quorum heuristic** against the published
  `mz_peaks`; `match_ms_hits` covers the portable matching if observed m/z lists
  (e.g. mzML) become available. MS is validation-only (M2); NMR is the spine that
  transfers to M4.
- The peak picker is noise-aware (`height = max(cy/mi floor, 5Žā)`, `prominence
  = 3Žā`, robust MAD Žā from signal-free windows). Calibrated on this ground truth;
  stable for noise multipliers 3ŌĆō8 (not a knife-edge fit).

# M2 ŌĆö Wire the tools into the multi-agent system

Integrates the M1 tools into BioAgent as a new **host-guest screening
(decision) phase**, following the existing architecture: deterministic tools run
FIRST and produce the verdicts; the LLM agents only *interpret and report* them
(the `Data_Discovery.py:67-68` pattern). Verdicts are never regenerated by the
model.

## What was added

| File | Role |
|------|------|
| `Tools/Decision_Phase.py` | Deterministic screening over an experiment set ŌåÆ `Screening_Verdicts` (mirrors the paper's `screening.py`, off-instrument). `Run_Screening` + `Screening_Hook` chatroom functions. |
| `Chatrooms/Decision_Making.py` | New chatroom modeled on `Data_Discovery.py`: runs `Run_Screening`, then the `Screening_Interpreter` agent narrates. |
| `Agents/Screening_Interpreter_Agent/` | Plate-level interpreter (schema `ScreeningInterpretationResponse`). |
| `Agents/NMR_Analysis_Agent/`, `Agents/MS_Analysis_Agent/` | Per-sample NMR / MS specialists (schemas `NMRAnalysisResponse` / `MSAnalysisResponse`). |
| `Utils/Pydantic_Schema.py` | +`ScreeningVerdict`, `ScreeningInterpretationResponse`, `NMRAnalysisResponse`, `MSAnalysisResponse` (additive; load-bearing `Step`/`PlanResponse` untouched). |
| `Agents/AgentFactory.py` | New schemas registered in the response-format ladder. |
| `Agents/agent_mapping.yml` | Three new agents registered. |
| `Agents/Focus_Area_Agent/system_message.yml` | Rewritten from biopharma/HPLC ŌåÆ supramolecular host-guest (NMR+MS+metal). |
| `BioAgent.py` | Screening context keys + `Screening_Mode` phase inserted after Focus Area (off by default, so the CSV pipeline is unchanged). |

## Validation (CPU, no GPU/vLLM)

- **Decision phase vs ground truth** (`Decision_Phase.screen_experiment_set`):
  processed-`1r` path reproduces **18/18** NMR, **18/18** MS, **18/18** overall
  (`REPLICATION`) verdicts; raw-FID path reproduces **18/18** overall.
- **Structural integration**: all modules `py_compile`; every new/edited YAML
  parses; `agent_mapping` Ōåö `config.response_format` Ōåö `AgentFactory` ladder are
  mutually consistent; all four new schemas instantiate; load-bearing schemas
  still import.

The LLM narration layer itself needs the vLLM/Qwen cluster to execute (M2's
end-to-end run) ŌĆö the deterministic verdicts it narrates are already proven here.
