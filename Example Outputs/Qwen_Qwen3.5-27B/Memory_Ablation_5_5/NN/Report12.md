#Abstract 

This report details the automated analysis of 11 chromatography runs to evaluate separation efficiency and product purity. The primary objective was to calculate the resolution ($R_s$) between adjacent peaks using a rigorous pipeline involving Savitzky-Golay smoothing, slope-based elution inference, and tangent skim integration for peak width determination. The analysis revealed a systemic failure across all evaluated runs, with every instance flagged as a 'CRITICAL PURITY FAILURE' due to minimum resolution values falling below the 1.5 threshold. While the visualization pipeline successfully identified co-eluting peaks and validated the smoothing algorithms, the quantitative results exhibited anomalous negative resolution values in several runs. These anomalies suggest potential upstream errors in peak detection or width calculation, likely stemming from low Signal-to-Noise Ratios (SNR) or algorithmic misinterpretation of noise as peaks. Consequently, while the conclusion of purity failure is robust, the specific magnitude of the resolution metrics requires further investigation into the peak definition phase.

#Introduction 

High-performance liquid chromatography (HPLC) is a critical technique for separating complex mixtures, where the resolution ($R_s$) between adjacent peaks serves as the primary metric for separation efficiency. In industrial and research settings, a minimum $R_s$ of 1.5 is often mandated to ensure baseline separation and guarantee product purity. This study focuses on the automated processing of high-frequency chromatography time-series data to assess compliance with this standard across 11 distinct experimental runs. The analysis workflow was designed to handle raw data containing pre-injection artifacts and sensor noise, requiring robust preprocessing steps including volume normalization and signal smoothing. The specific challenge addressed here involves the algorithmic inference of elution phases in the absence of explicit stage labels, relying instead on gradient slopes of the eluent concentration. By applying a strict resolution threshold, this analysis aims to programmatically identify runs that fail to meet purity criteria, thereby streamlining quality control processes. However, the reliability of such automated flagging depends heavily on the accuracy of the underlying peak detection and width calculation algorithms, particularly in the presence of variable signal-to-noise ratios.

#Methodology 

The data analysis followed a three-step protocol applied to the `chromatography_combined.csv` dataset. First, signal preprocessing involved normalizing the `volume_ml` axis by shifting values so the minimum volume per run equaled 0.0, explicitly excluding pre-injection negative volumes. The primary UV absorbance signal (`UV_1_280_mAU`) was smoothed using a Savitzky-Golay filter (window=11, polyorder=3) to mitigate high-frequency sensor noise. Elution phases were inferred by calculating the rolling slope of the eluent concentration (`Conc_B_%`); segments with a slope exceeding 0.5%/mL were identified as elution stages, with a fallback to monotonic increases in conductivity (`Cond_mS_cm`) if necessary. Peak detection was performed on the smoothed signal within these phases using a derivative-based approach, retaining only peaks with a Signal-to-Noise Ratio (SNR) greater than 3 and a height exceeding 5% of the maximum signal. 

Second, peak width at the baseline ($W_b$) was calculated using the 'tangent skim' method. This involved numerically locating inflection points on both sides of each peak, fitting tangents via linear regression, and determining their intersection with the baseline. Third, resolution ($R_s$) was calculated for all adjacent peak pairs using the formula $R_s = \frac{2(t_{R2} - t_{R1})}{W_{b1} + W_{b2}}$, utilizing raw shifted volume values to preserve precision. The minimum $R_s$ for each run was aggregated, and any run with a value below 1.5 was flagged as a 'CRITICAL PURITY FAILURE'. Finally, a consolidated visualization was generated to validate the detection logic and resolution annotations across all runs.

#Results and Discussion 

The analysis of the 11 chromatography runs yielded a uniform outcome: every run was flagged as a 'CRITICAL PURITY FAILURE' because the minimum calculated resolution ($R_s$) fell below the 1.5 threshold. As visualized in Figure 1, the chromatograms display smoothed UV traces with annotated $R_s$ values, where red annotations explicitly mark peak pairs failing the purity criterion. The visual consistency of the Savitzky-Golay smoothing and the clear demarcation of elution phases via the `Conc_B_%` gradient confirm that the preprocessing and phase inference steps functioned as intended. 

However, a deeper examination of the quantitative results reveals significant anomalies. While some runs exhibited low but plausible resolution values (e.g., Run 20: 0.00), others displayed physically impossible negative resolution values, such as -5205.43 for Run 17 and -232.31 for Run 16. Since $R_s$ is theoretically non-negative for distinct peaks, these extreme negative values indicate a breakdown in the upstream peak detection or width calculation logic. This failure is likely attributable to the heterogeneity in data quality observed across the runs. The analysis identified a subset of runs (1, 4, 16, 17) with low average SNRs (3.5–4.9), hovering near the detection threshold, which may have led to the identification of noise artifacts as peaks or the incorrect ordering of retention times ($t_{R2} < t_{R1}$). Conversely, Run 32 presented an outlier with 119 detected peaks and an exceptionally high SNR, suggesting a potential failure in peak filtering logic where noise was misidentified as a complex mixture. 

Despite these algorithmic irregularities, the conclusion that the separation efficiency is insufficient remains valid. The presence of co-eluting peaks, visually confirmed by the overlapping traces in Figure 1, supports the determination that the current experimental conditions do not achieve the required baseline separation. The negative $R_s$ values, while indicative of computational errors in the width calculation phase, do not negate the observation of poor separation; rather, they highlight the need for refined peak detection parameters, particularly for low-SNR signals, before precise efficiency metrics can be trusted.

#Conclusion 

The automated data analysis workflow successfully processed 11 chromatography runs to evaluate separation efficiency against a strict purity standard. The results unequivocally indicate that all analyzed runs failed to meet the minimum resolution requirement of 1.5, resulting in a 'CRITICAL PURITY FAILURE' status for the entire dataset. The visualization pipeline effectively communicated the presence of co-eluting peaks and validated the application of smoothing and elution inference algorithms. However, the quantitative resolution metrics were compromised by anomalous negative values in several runs, pointing to deficiencies in the peak detection and width calculation stages, particularly in low-SNR environments. Future work must focus on refining the peak filtering logic to distinguish true analytes from noise and correcting the algorithmic handling of peak boundaries to ensure physically meaningful resolution calculations. Until these methodological issues are resolved, the specific magnitude of the resolution failures should be interpreted with caution, although the overarching conclusion of inadequate separation remains robust.
