#Abstract 

This report details a rigorous data analysis workflow designed to detect non-physiological signal discontinuities in high-frequency chromatography time-series data. The primary objective was to distinguish between genuine instrument artifacts and expected process events, such as column blockages or gradient transitions, across four distinct column types (Q, SP, DEAE, CM). The methodology employed Asymmetric Least Squares (AsLS) baseline correction to normalize UV drift, followed by a volume-based rolling Z-score anomaly detection algorithm. Crucially, a cross-sensor correlation logic was implemented to filter out simultaneous UV and pressure spikes, classifying them as process events rather than artifacts. The analysis successfully identified 20 specific instrument artifacts, predominantly concentrated in Run 1, while effectively excluding pre-run equilibration data and post-injection stabilization windows. Visual validation confirmed the algorithm's ability to isolate optical noise from physical process transitions, ensuring high data integrity for downstream quality control.

#Introduction 

Chromatography data integrity is paramount for accurate process monitoring and quality control in biopharmaceutical manufacturing. However, high-frequency sensor data is often compromised by non-physiological discontinuities, such as optical noise or sensor glitches, which can be erroneously interpreted as process failures. Conversely, genuine process events, such as pressure spikes caused by column blockages, must be preserved to maintain a complete record of system behavior. This study addresses the challenge of differentiating between these two categories of anomalies within a complex dataset containing multiple column types and variable flow rates. The dataset presented significant preprocessing challenges, including erroneous variable duplication, negative volume entries representing pre-run equilibration, and a lack of explicit process stage annotations. The motivation for this analysis was to develop a robust, automated detection pipeline that leverages multi-sensor correlation and process stage inference to flag only true instrument artifacts, thereby enhancing the reliability of the chromatographic data for further analysis.

#Methodology 

The analysis followed a three-step protocol. First, data preprocessing involved filtering out rows with negative `volume_ml` to exclude pre-run equilibration artifacts, resulting in the removal of 32,007 rows. Erroneous variables ending in `_ml` were dropped. Asymmetric Least Squares (AsLS) baseline correction was applied to UV and conductivity signals using optimized parameters ($\lambda \approx 10^6\text{--}10^7$, $p \approx 2\times10^{-4}\text{--}2\times10^{-5}$) to correct for baseline drift. Process stages were inferred by calculating the first-order derivative of the Buffer B concentration (`Conc_B_%`); regions with a derivative greater than 5%/mL were classified as 'Wash/Transition', while others were 'Gradient/Elution'. Injection windows were identified via peak detection on the UV derivative, with a +2 mL stabilization buffer excluded from artifact detection. Second, anomaly detection utilized a volume-based rolling Z-score method. Data was resampled to a 0.1 mL grid, and a 5 mL rolling window was applied to calculate Z-scores. Points exceeding a Z-score of 3.0 were flagged. A cross-sensor correlation logic was then applied: if a UV spike occurred without a corresponding pressure or conductivity spike within a $\pm 0.5$ mL window, it was classified as an 'Instrument_Artifact'. Simultaneous spikes were classified as 'Process_Events'. Finally, the results were visualized for Run 1 (Column CM) to validate the detection logic against the raw signal and process context.

#Results and Discussion 

The analysis successfully identified four distinct column types (Q, SP, DEAE, CM) and revealed significant variation in run profiles. Runs 19, 20, and 21 were dominated by 'Gradient/Elution' events, whereas Runs 1, 2, 4, 16, 17, 30, and 32 were characterized by extensive stabilization phases. The anomaly detection algorithm identified 20 instances classified as 'Instrument_Artifacts'. Notably, these anomalies were heavily concentrated in Run 1 across both CM and DEAE columns, affecting UV, conductivity, and pressure sensors. As illustrated in Figure 1, the visualization for Run 1 (Column CM) confirms the efficacy of the detection logic. The plot displays the baseline-corrected UV absorbance (black line), pressure differential (blue line), and Buffer B gradient (light gray line). Red scatter points, denoting 'Instrument_Artifacts', appear as isolated, sharp vertical deviations in the UV trace that lack corresponding pressure anomalies, validating the algorithm's ability to distinguish optical noise from physical process events. Conversely, blue points indicate 'Process_Events' where UV and pressure spikes coincide, correctly excluding these from the artifact list. The light gray shaded region highlights the post-injection stabilization window, which was successfully excluded from the detection logic. While the data quality appears high with minimal noise in baseline regions, the presence of negative volume values in the artifact summary table suggests a potential boundary inconsistency in the exclusion logic that warrants further investigation. The high volume of stabilization data in most runs implies that rolling window parameters require careful tuning to avoid false positives during low-variance periods.

#Conclusion 

The implemented data analysis workflow effectively distinguishes between non-physiological instrument artifacts and genuine process events in chromatography data. By integrating AsLS baseline correction, volume-based rolling Z-scores, and cross-sensor correlation logic, the study successfully isolated 20 instrument artifacts, primarily within Run 1, while preserving valid process transitions. The visual validation in Figure 1 demonstrates that the algorithm correctly identifies isolated UV spikes as artifacts and simultaneous UV-pressure spikes as process events. The exclusion of pre-run equilibration and post-injection stabilization windows further refined the detection accuracy. However, the analysis highlighted the need for careful tuning of rolling window parameters to manage false positives during long stabilization phases and addressed minor inconsistencies regarding negative volume boundaries. This methodology provides a robust framework for automated quality control in complex chromatographic datasets, ensuring that data integrity is maintained for critical decision-making processes.
