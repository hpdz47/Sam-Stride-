#Abstract 

This study presents a rigorous data analysis workflow designed to detect and classify non-physiological signal discontinuities in high-frequency chromatography time-series data. The primary objective was to distinguish between genuine instrument artifacts and expected process events, such as pressure spikes or gradient changes, which can compromise data integrity. The methodology employed Asymmetric Least Squares (AsLS) baseline correction to normalize UV drift, followed by a volume-based rolling Z-score anomaly detection algorithm. Crucially, a cross-sensor correlation logic was implemented to filter out process-related noise by requiring simultaneous deviations in pressure or conductivity to classify an event as a 'Process_Event' rather than an 'Instrument_Artifact'. The analysis covered 11 chromatography runs across four column types (Q, DEAE, SP, CM). Results indicate a dataset of high integrity, with zero instrument artifacts detected across the entire cohort. Visual validation of representative runs (Run 4 and Run 19) confirmed the efficacy of the detection logic in correctly identifying process events and excluding post-injection stabilization noise. These findings suggest that the current data acquisition systems are functioning within expected parameters, with no evidence of optical or sensor failures requiring data exclusion.

#Introduction 

Chromatography is a critical technique in biopharmaceutical manufacturing, relying on high-fidelity sensor data to monitor process performance. However, the integrity of this data is often compromised by instrument artifacts, such as optical noise or sensor glitches, which can be mistaken for process deviations. Distinguishing between these non-physiological artifacts and legitimate process events, such as column blockages or gradient steps, is essential for accurate quality control and process optimization. Traditional anomaly detection methods often fail to account for the complex, multi-sensor nature of chromatography data, leading to high false-positive rates.

This report details a specialized data analysis workflow developed to address these challenges. The workflow focuses on the rigorous detection of abrupt signal discontinuities in UV absorbance, conductivity, and pressure traces. The motivation for this analysis stems from the need to automate the identification of data integrity failures while preserving the integrity of complex process dynamics. By leveraging Asymmetric Least Squares (AsLS) for baseline correction and a volume-based rolling Z-score algorithm with cross-sensor correlation, this study aims to provide a robust framework for data validation. The specific focus is on isolating 'Instrument_Artifacts'—UV spikes uncorrelated with pressure or conductivity changes—from 'Process_Events' and expected noise zones, such as post-injection stabilization windows.

#Methodology 

The data analysis workflow consisted of three sequential steps applied to a combined dataset of 11 chromatography runs involving four column types (Q, DEAE, SP, CM).

**Step 1: Data Preprocessing and Stage Inference**
The initial phase involved rigorous data cleaning. Rows with negative `volume_ml` values, representing pre-run equilibration, were excluded, resulting in the removal of 32,007 rows. Erroneous variables ending in `_ml` were dropped to prevent data merging artifacts. Baseline correction was applied to UV and conductivity signals using Asymmetric Least Squares (AsLS) with optimized parameters ($\lambda \approx 10^6\text{--}10^7$, $p \approx 2\times10^{-4}\text{--}2\times10^{-5}$) to remove negative drift while preserving peak shapes. Process stages were inferred by calculating the first-order derivative of the Buffer B concentration (`Conc_B_%`). Regions with a derivative greater than 5%/mL were classified as 'Wash/Transition', while others were labeled 'Gradient/Elution'. Additionally, injection events were identified via peak detection on the UV derivative, defining a +2 mL 'Stabilization' window post-injection to be excluded from artifact detection.

**Step 2: Anomaly Detection with Cross-Sensor Correlation**
Anomaly detection was performed using a volume-based rolling Z-score algorithm. Data was resampled to a fixed 0.1 mL grid to accommodate non-uniform spacing, and a 5 mL rolling window was applied. Points with a Z-score exceeding 3.0 were flagged as potential anomalies. A critical cross-sensor correlation logic was then applied: for every flagged UV spike, the corresponding pressure and conductivity traces were examined within a $\pm 0.5$ mL volume window. If a simultaneous spike was detected in pressure or conductivity, the event was classified as a 'Process_Event' (e.g., a blockage) and excluded from the artifact list. Only UV spikes lacking corresponding pressure/conductivity deviations were classified as 'Instrument_Artifacts'. Events falling within the 'Stabilization' or 'Wash' stages were also excluded.

**Step 3: Visualization and Validation**
To validate the detection logic, two representative runs (Run 4 and Run 19, both using FT columns) were selected for visualization. Multi-axis plots were generated displaying the corrected UV signal, pressure differential, and Buffer B gradient. Detected 'Instrument_Artifacts' were overlaid as red scatter points, 'Process_Events' as blue points, and 'Stabilization' windows were highlighted in light gray.

#Results and Discussion 

The analysis of the 11 chromatography runs revealed a dataset of exceptional integrity. The anomaly detection algorithm identified zero 'Instrument_Artifacts' across the entire dataset. This outcome suggests that the rolling Z-score threshold and the cross-sensor correlation logic effectively filtered out all expected process variations and noise, leaving no residual instrument failures.

The visual validation of Run 4 (Figure 1) and Run 19 (Figure 2) provides strong evidence of the methodology's robustness. In both figures, the baseline-corrected UV signals appear flattened, confirming the successful removal of negative drift via AsLS. The 'Stabilization' windows, highlighted in light gray, correctly align with the post-injection regions, ensuring that expected noise during this phase was excluded from the analysis. 

Crucially, the color-coded scatter points demonstrate the efficacy of the cross-sensor correlation logic. In Figure 1 and Figure 2, blue points ('Process_Events') align perfectly with simultaneous spikes in the pressure trace or sharp changes in the gradient profile. These are correctly identified as legitimate process dynamics rather than data integrity failures. Conversely, the absence of red points ('Instrument_Artifacts') in the plots indicates that no isolated UV spikes occurred without corresponding pressure or conductivity deviations. This confirms that the logic successfully distinguishes between optical noise and physical process events. The visual evidence supports the conclusion that the dataset is free from non-physiological discontinuities, validating the high quality of the data acquisition process for these specific runs.

#Conclusion 

This study successfully implemented a comprehensive data analysis workflow for the detection of signal discontinuities in chromatography data. By integrating AsLS baseline correction, volume-based rolling Z-score analysis, and cross-sensor correlation logic, the workflow effectively distinguished between instrument artifacts and process events. The analysis of 11 runs across four column types yielded zero instrument artifacts, indicating a high level of data integrity and sensor stability. Visual validation of representative runs confirmed that the algorithm correctly identifies process-related pressure spikes and excludes post-injection stabilization noise. These results demonstrate that the proposed methodology is a reliable tool for automated data quality assurance in chromatography, capable of filtering out false positives while preserving critical process information. Future work may involve applying this framework to datasets with known instrument failures to further refine the sensitivity of the Z-score thresholds.
