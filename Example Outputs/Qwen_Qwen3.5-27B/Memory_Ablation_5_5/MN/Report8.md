#Abstract 

This report details the analysis of 1.08 million rows of high-frequency chromatographic data across 11 experimental runs and four column types (CM, DEAE, Q, SP). The primary objective was to algorithmically identify co-eluting peaks and assess run-to-run variability using Resolution Factors ($R_s$) calculated via peak width at the baseline. The analysis employed Adaptive Smoothing Least Squares (AsLS) for dynamic baseline correction to handle pre-run noise and negative volume artifacts. Results indicate a systemic failure in separation efficiency, with the majority of peak pairs exhibiting negative $R_s$ values, signifying extensive co-elution. Consequently, the Co-elution Index could not be computed due to the absence of high-resolution reference runs required for statistical thresholding. Visualization of the worst-performing run confirms that insufficient gradient steepness is the primary driver of poor resolution. Furthermore, variability assessment via Relative Standard Deviation (RSD) of the worst-resolved peaks highlights significant inconsistencies in column performance, necessitating targeted maintenance or gradient optimization.

#Introduction 

Chromatographic separation efficiency is critical for ensuring product purity, particularly in bioprocessing where co-elution of impurities can compromise downstream applications. This study focuses on the rigorous evaluation of UV absorbance time-series data ($UV_{280}$ and $UV_{260}$) against elution volume and eluent concentration ($Conc_B$) to detect poor resolution. The dataset presented significant challenges, including 75% missingness in sample identifiers and 11.5% missingness in fraction IDs, requiring aggregation by run and column. The analysis aimed to calculate the Resolution Factor ($R_s$) using peak widths determined by linear interpolation against an AsLS-corrected baseline, targeting a threshold of $R_s < 1.5$ to identify co-elution. A key component of this workflow was the derivation of local gradient slopes within co-eluting valleys to determine the minimum steepness required for baseline separation. Additionally, the study sought to compute a Co-elution Index by evaluating the $UV_{260}/UV_{280}$ ratio against a statistically derived threshold to detect nucleic acid impurities. However, the success of this index relies on the presence of high-resolution runs, which were anticipated to be scarce given the dataset's characteristics.

#Methodology 

The data analysis workflow comprised three primary steps. First, signal preprocessing involved loading the raw time-series data and removing erroneous columns (e.g., `Conc_B_ml`, `pH_ml`) that were misaligned duplicates of the volume axis. Adaptive Smoothing Least Squares (AsLS) baseline correction was applied to $UV_{280}$ and $UV_{260}$ signals, utilizing negative volume ranges to estimate the baseline before filtering out pre-run data. Peaks were detected with a minimum height threshold (>10 mAU) to exclude noise, and 'waste zones' (regions with missing `Fraction_unique_ID`) were flagged. Second, the Resolution Factor ($R_s$) was calculated for adjacent peak pairs using the formula $R_s = 2 * (V_{R2} - V_{R1}) / (W_{b1} + W_{b2})$, where peak widths were determined via linear interpolation at the baseline. Co-eluting pairs ($R_s < 1.5$) were identified, and local gradient slopes were derived within these valleys. The Co-elution Index was intended to flag regions where $R_s < 1.5$ and the $UV_{260}/UV_{280}$ ratio exceeded a threshold derived from high-resolution runs ($R_s > 2.0$). Third, run-to-run variability was assessed by calculating the Relative Standard Deviation (RSD) of the minimum $R_s$ values for each column. Visualizations were generated to highlight the absolute worst-performing run and summarize column variability.

#Results and Discussion 

The analysis revealed a critical lack of separation across the dataset. Step 2 results indicated that the minimum $R_s$ values were predominantly negative (e.g., -15.4 for Run 17 SP, -50.0 for Run 20 CM), suggesting that peak boundaries overlapped significantly or that peak width calculations resulted in $V_{R2} < V_{R1}$. This systemic co-elution prevented the identification of any 'high-resolution runs' ($R_s > 2.0$), rendering the calculation of the statistical threshold for the Co-elution Index impossible. Consequently, the $UV_{260}/UV_{280}$ ratio could not be used to assess nucleic acid impurities in this specific dataset.

Figure 1 illustrates the absolute worst-performing run, where the AsLS baseline correction successfully stabilized the signal near zero prior to the first peak. The plot clearly delineates waste zones and highlights a co-elution valley between two major peaks where $R_s < 1.5$. The overlay of the $Conc_B$ gradient reveals a shallow local slope in this valley, providing direct evidence that the current gradient steepness is insufficient to achieve baseline separation. This confirms that the primary failure mode is driven by gradient profile limitations rather than column capacity or baseline instability.

Figure 2 presents the Run-to-Run Variability Assessment, plotting the RSD of the worst-resolved peak pair for each column. The varying heights of the bars indicate differing degrees of fluctuation in separation performance across the four columns. Higher RSD values suggest inconsistent column performance, potentially due to aging, packing heterogeneity, or process parameter drift. This visualization effectively aggregates complex time-series data into a comparable metric, identifying which columns require maintenance. The analysis confirms that while peak detection algorithms functioned correctly, the chromatographic process under the tested conditions failed to resolve components, necessitating a re-evaluation of gradient profiles and column health.

#Conclusion 

The data analysis successfully identified a systemic failure in chromatographic resolution across all 11 runs and four column types. The calculation of Resolution Factors revealed extensive co-elution, with minimum $R_s$ values frequently falling below zero. This poor separation prevented the computation of the Co-elution Index, as no high-resolution reference runs were available to establish a statistical threshold for nucleic acid impurity detection. The visualization of the worst-performing run (Figure 1) confirmed that insufficient gradient steepness is the primary cause of co-elution, while the variability assessment (Figure 2) highlighted significant inconsistencies in column performance. Future optimization efforts must focus on increasing the local gradient slope in critical separation regions and addressing column heterogeneity to achieve the target $R_s \geq 1.5$. Once baseline separation is achieved, the Co-elution Index can be successfully deployed to monitor product purity.
