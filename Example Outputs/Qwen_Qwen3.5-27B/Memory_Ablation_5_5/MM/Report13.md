#Abstract 

This report evaluates the reproducibility of a High-Performance Liquid Chromatography (HPLC) system across 11 runs using four distinct column types (CM, DEAE, Q, and SP). The primary objective was to quantify the Coefficient of Variation (CV) of the main product peak retention times against a 2.0% benchmark defined by USP <621>. The analysis involved rigorous data preprocessing, including flow rate normalization, Asymmetric Least Squares (AsLS) baseline correction, and IQR-based outlier filtering. Results indicate a critical failure in the reproducibility assessment for the CM column, which exhibited a CV of 60.3%, significantly exceeding the acceptable threshold. Furthermore, the analysis revealed insufficient data for the DEAE, Q, and SP columns due to aggressive outlier filtering that removed all valid runs for these strata. These findings suggest potential systemic errors in flow rate normalization or baseline correction logic, necessitating a review of the data processing pipeline before valid reproducibility metrics can be established.

#Introduction 

Chromatographic system reproducibility is a fundamental requirement for ensuring data integrity in analytical workflows. This study focuses on assessing the consistency of main product peak retention times across multiple HPLC runs. The dataset comprises 11 runs utilizing four different chromatography column types: CM, DEAE, Q, and SP. A significant challenge in this analysis is the absence of an explicit time column in the raw data, requiring the derivation of retention times from volume and flow rate variables. Additionally, the dataset presents heterogeneity in column types and potential data quality issues, such as missing sample codes and potential unit conversion errors.

The motivation for this analysis is to determine if the current chromatographic system meets the industry standard for reproducibility, defined as a Coefficient of Variation (CV) of less than 2.0% for retention times. Achieving this benchmark is critical for validating the system's reliability for future quantitative analyses. The workflow employed a stratified approach to account for column-specific variations, applying robust statistical methods to filter outliers and estimate baseline variance. This report details the methodology used to process the raw UV signal data and discusses the unexpected results regarding system performance and data sufficiency.

#Methodology 

The data analysis followed a three-step plan designed to transform raw signal data into robust reproducibility metrics. 

Step 1 involved Data Preprocessing, Baseline Correction, and Peak Detection. Raw data from `Chromatography_Combined.csv` was filtered to exclude pre-injection data (`volume_ml` <= 0) and invalid flow rates (`System_flow_ml_min` < 0.01 ml/min). Retention time was calculated by normalizing volume to time, propagating the last valid flow rate for intervals with zero flow. Asymmetric Least Squares (AsLS) baseline correction was applied to the `UV_1_280_mAU` signal, with parameters optimized via grid search to minimize residuals while preserving peak height. Peaks were detected using a Signal-to-Noise (S/N) ratio threshold of >3. The 'Main Product Peak' was identified by prioritizing the 'Elution' chromatography stage or, in its absence, the peak with the maximum area.

Step 2 focused on Stratified Baseline Variance Estimation and Outlier Filtering. Data was grouped by column type. Empirical baseline variance was estimated for retention times within each stratum. Outliers were identified and removed using the Interquartile Range (IQR) method (Q1 - 1.5*IQR, Q3 + 1.5*IQR), rather than Grubbs' test, to accommodate small sample sizes.

Step 3 performed Reproducibility Quantification and Benchmarking. For each column stratum, the number of remaining runs was counted. If fewer than 5 runs remained, the column was flagged as 'Insufficient Data'. Otherwise, the mean, standard deviation, and CV were calculated. The CV was compared against the 2.0% benchmark to determine a 'Pass' or 'Fail' status. Visualizations were generated to display the distribution of retention times relative to the benchmark limits.

#Results and Discussion 

The analysis results highlight significant discrepancies in data quality and system performance across the different column types. As visualized in Figure 1, the reproducibility assessment for the CM column resulted in a definitive 'Fail' status. The calculated Coefficient of Variation (CV) for the CM column was 60.3%, based on 11 runs with a mean retention time of 126.2 minutes. This value drastically exceeds the 2.0% benchmark, indicating severe variability in the system's performance for this specific column type. The boxplot in Figure 1 illustrates this high variability, with the distribution of retention times extending well beyond the shaded benchmark regions.

A critical issue identified during the analysis was the inconsistency in data processing for the CM column. Initial aggregated statistics suggested an anomalously high mean retention time (~465,000 units), which was later corrected to ~109 minutes after outlier removal in Step 2, and finally to 126.2 minutes in Step 3. This fluctuation suggests potential errors in the flow rate normalization step or inconsistencies in the application of baseline correction and outlier filtering. The high CV implies that the system is not stable for the CM column under the current operating conditions.

Furthermore, the analysis revealed a complete lack of valid data for the DEAE, Q, and SP columns. Although initial data counts suggested the presence of runs for these columns (1 run for DEAE and 4 for Q), the IQR-based outlier filtering in Step 2 removed all valid runs, resulting in a count of zero for the final reproducibility calculation. Consequently, these columns were flagged as 'Insufficient Data'. This outcome suggests that the outlier filtering parameters may have been overly stringent for these specific strata, or that the raw data for these columns contained systematic errors that were misidentified as outliers. The inability to calculate a CV for three out of four column types prevents a comprehensive assessment of the system's overall reproducibility. The visual representation in Figure 1 confirms the absence of data boxes for DEAE, Q, and SP, reinforcing the conclusion that the current data processing pipeline requires optimization to retain valid data points for these columns.

#Conclusion 

In conclusion, the current data analysis workflow failed to validate the reproducibility of the HPLC system. The CM column demonstrated unacceptable variability with a CV of 60.3%, far exceeding the 2.0% benchmark. More critically, the analysis yielded insufficient data for the DEAE, Q, and SP columns due to the total removal of runs during the outlier filtering stage. These results indicate that the system is not currently fit for purpose and that the data processing methodology requires immediate revision. Specifically, the flow rate normalization logic and the stringency of the IQR outlier filtering parameters must be reviewed to address the unit conversion anomalies and data loss observed. Future work should focus on recalibrating the baseline correction parameters and adjusting the outlier detection thresholds to ensure that valid data is retained for statistical evaluation across all column types. Without these corrections, no reliable conclusions regarding system reproducibility can be drawn.
