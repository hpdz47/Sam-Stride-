#Abstract 

This report details the quantitative analysis of High-Performance Liquid Chromatography (HPLC) data to assess product purity, abundance, and system stability. The workflow addressed significant data quality challenges, including non-physical negative volume intervals and a systematic negative baseline drift of approximately -7.95 mAU. By implementing Asymmetric Least Squares (AsLS) baseline correction and rigorous pre-run filtering, the study successfully normalized high-frequency UV signals. The analysis stratified results by run number and column type, calculating integrated peak areas to derive 260/280 purity ratios and relative abundance metrics. Visual inspection of the processed data confirmed effective gradient alignment and baseline normalization, establishing a robust foundation for assessing chromatographic consistency and identifying potential system anomalies.

#Introduction 

The primary objective of this data analysis was to process complex HPLC datasets to generate accurate metrics regarding protein abundance and purity. The dataset presented specific challenges, including the presence of pre-run intervals with negative volume values (down to -33.8 ml) and a consistent negative baseline drift in UV signals. These artifacts, if uncorrected, would lead to erroneous integration results and flawed purity assessments. Furthermore, the dataset exhibited incomplete metadata, with approximately 75% of rows lacking specific sample codes and 11.5% missing fraction identifiers, necessitating a dual-strategy approach: aggregating data by run and column for general trends while reserving detailed sample-specific analysis for the subset with complete metadata. The focus was on utilizing integrated peak areas rather than peak heights to ensure robustness against peak shape variations, specifically targeting the calculation of the 260/280 UV ratio to flag potential impurities (ratios > 0.6). Additionally, the analysis aimed to validate system stability by monitoring pressure traces and gradient alignment during both collection and waste phases.

#Methodology 

The analysis followed a three-step protocol. First, signal preprocessing involved loading the raw chromatography data and filtering out all rows with negative `volume_ml` to exclude non-physical pre-run intervals. Asymmetric Least Squares (AsLS) baseline correction was applied to the `UV_1_280_mAU` and `UV_2_260_mAU` signals using parameters $\	ext{lambda} = 1,000,000$ and $p = 0.0001$ to effectively remove the observed negative drift. Data was stratified by `run_no` and `column`, with rows missing `Fraction_unique_ID` retained separately for system stability monitoring. Second, peak integration was performed using the trapezoidal rule on the corrected signals against cumulative volume. Purity ratios (`Ratio_260_280`) were calculated as the ratio of integrated areas at 260 nm to 280 nm, with safety checks to prevent division by zero. Abundance was normalized both by total run area and by fraction volume. Third, system stability and gradient alignment were visualized by plotting corrected UV signals alongside the gradient profile (`Conc_B_%`) and pressure traces (`DeltaC_pressure_MPa`). This step included the detection of retention time outliers using Interquartile Range (IQR) methods and the identification of pressure anomalies exceeding three standard deviations from the mean.

#Results and Discussion 

The preprocessing stage successfully addressed the data quality issues. The filtering of negative volume intervals removed between 60 and 6,360 rows per run, ensuring that only physical flow data was analyzed. The application of AsLS correction effectively normalized the baseline, as evidenced by the alignment of pre-peak and post-peak regions with the zero baseline in Figure 1. However, a notable discrepancy was observed in the data retention for system monitoring; while most runs retained thousands of rows for stability assessment, runs 18 through 32 showed zero rows retained for monitoring, suggesting a potential data inconsistency or exclusion logic error for these specific experiments that requires verification. 

Figure 1 provides a comprehensive visual assessment of the system stability and gradient alignment across all runs and columns. The multi-panel grid clearly demonstrates that the UV peaks align temporally with the expected elution windows defined by the inflection points of the `Conc_B_%` gradient profile. The distinct step changes in the gradient are visible, and the corrected UV signals show consistent peak shapes across different columns within the same run, supporting the validity of the gradient alignment logic. While the static image does not explicitly annotate pressure anomalies, the visual smoothness of the baseline and the consistency of peak apexes relative to the volume axis suggest that the system remained stable during the collection phases. The regions corresponding to missing `Fraction_unique_ID` (waste phases) are distinguishable, confirming that the separation of collection and waste data was successful. These visual results confirm that the preprocessing pipeline has normalized the data sufficiently to allow for reliable quantitative integration and subsequent purity ratio calculations, although the gap in monitoring data for later runs remains a limitation for a complete system stability assessment.

#Conclusion 

The data analysis workflow successfully transformed raw, noisy HPLC data into a robust dataset suitable for quantitative assessment of protein purity and abundance. The implementation of AsLS baseline correction and negative volume filtering effectively resolved the primary data artifacts, enabling accurate trapezoidal integration of UV signals. The visual evidence from Figure 1 confirms that the gradient elution was consistent and that the baseline correction preserved peak integrity. While the analysis highlighted a potential data gap regarding system monitoring for runs 18 through 32, the overall methodology proved effective for the majority of the dataset. The resulting metrics, including 260/280 purity ratios and normalized abundance values, provide a reliable basis for evaluating product quality and process consistency. Future work should address the data inconsistency in the later runs to ensure comprehensive system stability monitoring across the entire experimental series.
