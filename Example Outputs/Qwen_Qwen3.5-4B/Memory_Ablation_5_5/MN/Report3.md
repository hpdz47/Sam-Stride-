#Abstract 

This report details a rigorous data integrity audit performed on a high-resolution HPLC chromatography dataset intended for machine learning applications. The primary objective was to evaluate whether the dataset meets strict biopharmaceutical standards regarding signal quality, completeness, and unit consistency. Analysis revealed severe data integrity failures, including a 65.6% missingness rate in critical variables such as `Conc_B_ml` and `pH_ml`, and the presence of negative value artifacts in UV absorbance columns, indicating data entry errors or baseline noise. Furthermore, a significant 'Modeling Feasibility Gap' was identified, where only 24.32% of rows contain both `chromatography_stage` and `Sample_Code`, rendering sample-specific modeling infeasible for the majority of the data. The methodology involved a three-step process: a completeness audit, dual-validation artifact detection, and the derivation of a high-completeness feature set. Based on these findings, the 'Fit for Use' dataset was constructed by excluding variables with missingness exceeding 95% and retaining only those with valid, non-negative units. Consequently, the final usable dataset comprises 1,084,069 rows utilizing a reduced variable set of `run_no`, `UV_1_280_mAU`, `UV_2_260_mAU`, `Cond_mS_cm`, `Conc_B_%`, and `pH_pH`. This report concludes that while a high-quality feature set can be derived, the dataset cannot currently support granular sample tracking without aggressive data reconstruction.

#Introduction 

In the context of biopharmaceutical manufacturing, the reliability of chromatographic data is paramount for downstream machine learning and process optimization. However, datasets derived from high-resolution fraction-level time-series often suffer from critical integrity issues that compromise model reliability. This analysis focuses on a dataset containing variables such as concentration, pH, UV absorbance, and process metadata, which exhibits severe completeness gaps and unit validity failures. Specifically, key variables like `Conc_B_ml` and `pH_ml` demonstrate missingness rates exceeding 65%, while unit confusion has led to negative absorbance values in columns labeled with `_ml`, suggesting mislabeling between volume and absorbance metrics. The motivation for this analysis was to quantify these failures and determine if the dataset is 'fit for use' by strictly adhering to biopharmaceutical standards. The goal was to identify specific data integrity failures—such as missingness exceeding 65% in key variables, unit mislabeling, and missing process metadata—that would render predictive modeling unreliable. The analysis aims to provide a quantified decision on modeling feasibility and to guide the exclusion of corrupted variables rather than introducing bias through aggressive imputation, ensuring that only variables with >95% completeness and valid units are utilized for feature engineering.

#Methodology 

The data analysis workflow was structured into three distinct steps to ensure a comprehensive evaluation of the dataset's integrity. Step 1 involved a Data Integrity and Completeness Audit. This phase calculated exact row percentages for missingness and unit validity across all variables, including `run_no`, `chromatography_stage`, `Sample_Code`, and various concentration and pH metrics. The objective was to measure the extent of failure without fixing the data, explicitly defining the 'fit for use' dataset boundaries. Step 2 focused on Dual-Validation Artifact Detection. This step analyzed signal integrity for validated variables, specifically `UV_1_280_mAU` and `UV_2_260_mAU`, to detect negative value artifacts. Additionally, a correlation analysis was performed between `UV_1_2_280_mAU` and `volume_ml` to verify potential unit mislabeling, validating the plan's decision to filter rather than impute. Step 3 involved the High-Completeness Feature Set Derivation. This phase constructed the final dataset by filtering for variables meeting strict biopharmaceutical standards: >95% completeness and valid units (non-negative). Variables failing these criteria were excluded, and only `run_no` was used for identification to ensure consistency. The final output included a summary count of retained rows and an explicit statement of exclusion criteria, concluding with the calculated percentage of rows supporting sample-specific modeling.

#Results and Discussion 

The visualizations presented in Figure 1 provide critical evidence regarding the data integrity of the UV absorbance metrics. The histogram, titled 'Distribution of UV Absorbance (mAU)', plots the frequency of `UV_1_280_mAU` and `UV_2_260_mAU` values. A distinct vertical line at 0 highlights the presence of negative value artifacts, as the distribution tails extend below the zero baseline. This indicates signal noise or data entry errors where absorbance should be non-negative. The scatter plot, titled 'UV_1_280_mAU vs Volume (mL)', reveals a strong positive correlation between the absorbance signal and the cumulative eluent volume. This correlation serves as methodological evidence for detecting unit mislabeling; specifically, it suggests that the variable `UV_1_280_ml` (which has a mean of ~126.5 and a range matching volume) was likely incorrectly labeled as absorbance, as true absorbance should not scale linearly with total volume in this manner. These visualizations effectively highlight the critical data integrity issues identified in the dataset report: the existence of negative absorbance outliers and the high likelihood of unit confusion between volume and absorbance metrics. Consequently, the analysis confirms the necessity of excluding `_ml` variants and retaining only `_mAU` variables for the 'Fit for Use' dataset. Furthermore, the text-based analysis confirms that only 1,084,069 rows were retained after applying strict thresholds, utilizing a reduced variable set of `run_no`, `UV_1_280_mAU`, `UV_2_260_mAU`, `Cond_mS_cm`, `Conc_B_%`, and `pH_pH`. A critical finding is the 'Modeling Feasibility Gap,' where only 24.32% of rows possess both `chromatography_stage` and `Sample_Code`, rendering sample-specific modeling infeasible for the vast majority of the dataset. The exclusion of negative values in physical quantities like UV absorbance confirms the presence of data entry errors or baseline noise artifacts, validating the plan's decision to filter rather than impute. While the analysis effectively isolated high-quality features, the remaining 75.7% missingness in process metadata highlights a significant limitation: the dataset cannot currently support granular sample tracking without aggressive data reconstruction or exclusion of the majority of the data volume.

#Conclusion 

The comprehensive data integrity audit concludes that the original chromatography dataset is not fit for use in its current state for broad machine learning applications due to severe completeness and unit validity failures. The presence of negative absorbance values and the linear correlation between absorbance and volume confirm critical unit mislabeling errors, necessitating the exclusion of variables such as `Conc_B_ml`, `pH_ml`, and `UV_1_280_ml`. Additionally, the high missingness rates in key variables and the absence of sample metadata in 75.7% of the rows create a substantial 'Modeling Feasibility Gap,' limiting the utility of the data to run-level analysis rather than sample-specific modeling. However, the analysis successfully derived a robust 'Fit for Use' feature set comprising 1,084,069 rows with valid, non-negative units and >95% completeness. This reduced dataset, utilizing variables like `UV_1_280_mAU`, `UV_2_260_mAU`, `Cond_mS_cm`, `Conc_B_%`, and `pH_pH`, provides a reliable foundation for feature engineering and predictive modeling at the run level. Future efforts should focus on data reconstruction strategies to address the missing process metadata and validate the physical consistency of the remaining variables to fully realize the potential of this dataset.
